Pre-Processing Confidence Score v1.1 · Evidence-Based · Four-Block + Per-Artifact Views
Four-block evidence-based pre-processing confidence chain with separate verification_status field. CBMI does not perform processing — chain evaluates artifacts and declared/optional processing provenance. Tiered evidence with graceful degradation when reports are not uploaded. Path-aware indicators evaluate only when their applies_to_paths matches the operator-declared workflow. CP-related quality reported as verification_status rather than gating the score — surveys can be high-quality without check points.
Computed entirely from pre-processing artifacts (geotagged images, GCP coord file, CP coord file) + processing manifest + optional processing report + external sources. Does not read drone_score, base_station_score, gcp_score, or check_point_score outputs at runtime.
Artifacts (geotags + GCP coords + CP coords)→+ Manifest→+ optional Report→Indicators (path-aware, tiered evidence)→Building Blocks→pre_processing_score + verification_status
Core — pre_processing_score
Single scalar score from four blocks. Evidence-based — CBMI does not process; chain evaluates artifacts + declared provenance.
pre_processing_score
Pre-Processing Confidence Score
0.35*reference_frame_score + 0.30*geotag_integrity_score + 0.25*gcp_coord_trust_score + 0.10*survey_design_score
aggregator: block-of-blocks; within-block weighted averages with redistribution when evidence absent or path-N/A
⊘ GLOBAL GATE: Sets pre_processing_score = 0. The named flag fires with severity CATASTROPHIC.
∅ NULL: No null state. Score always computes from the four scoring blocks. The CP situation is reported via verification_status, not as a null.
verification_status field (separate, not score-gating):
Separate categorical field reporting CP-related quality. VALUES: VERIFIED, UNVERIFIED_NO_CPS, UNVERIFIED_INSUFFICIENT_CPS, UNVERIFIED_CP_CLUSTERED, UNVERIFIED_CP_NOT_INDEPENDENT, UNVERIFIED_CP_TRUST_LOW. NEVER score-gating. PP_NO_INDEPENDENT_VERIFICATION flag (CRITICAL-informational) fires when verification_status != VERIFIED. Reasoning: surveys can be high-quality without check points; absence of verification does not invalidate the survey quality.
Evidence-based confidence score for pre-processing outputs. CBMI does NOT perform processing — this chain evaluates artifacts (geotagged images, GCP coord file, CP coord file) and a declared processing manifest. Optional processing report deepens diagnostic depth. Runtime-independent of drone_score, base_station_score, gcp_score, check_point_score — does not read their outputs. Cross-subsystem coupling (e.g., "CP reliability degraded by base risk") belongs in future capture_score.
Paths supported: AUTONOMOUS / PUBLISHED_BENCHMARK / CORS / CLOUD / CUSTOMER_SUPPLIED / LOCAL_BASE_PPK (path-aware indicators only evaluate when their applies_to_paths matches)
Level 1 — Building Blocks
Four scoring blocks. CP-related quality reported via verification_status field, NOT as a fifth block.
reference_frame_score
Reference Frame Integrity
weight 0.35 in pre_processing_score
Q: Are all artifacts in the same, declared, correct reference frame?  ·  Owner: Pre-processing software setup + processing manifest  ·  If low: Verify CRS/datum/geoid/projection/units declarations match project requirement
aggregator: weighted average of indicator scores indicators: ref_frame_declared_score(0.30), geoid_model_declared_score(0.20), height_mode_declared_score(0.15), projection_declared_score(0.10), output_crs_metadata_score(0.10), units_declared_score(0.05), customer_coord_crs_declared_score(0.05), localization_disclosed_score(0.03), provenance_consistency_score(0.02)
⊘ reference_frame_score = 0. Zeros pre_processing_score via block-of-blocks.
geotag_integrity_score
Geotag Integrity
weight 0.3 in pre_processing_score
Q: Are the geotagged images trustworthy as photogrammetry input?  ·  Owner: Drone PPK processing software + base/CORS data  ·  If low: Reprocess with correct base/CORS pairing; verify per-image solution status
aggregator: weighted average of indicator scores (with weight redistribution when evidence absent) indicators: base_pairing_score(0.25), geotag_solution_quality_score(0.20), geotag_completeness_score(0.15), geotag_baseline_score(0.10), session_overlap_score(0.10), overlap_texture_score(0.08), antenna_pco_score(0.05), sensor_metadata_consistency_score(0.03), cors_data_continuity_score(0.02), time_sync_residual_score(0.01), monsoon_flight_artifact_score(0.01)
gcp_coord_trust_score
GCP Coordinate Trust
weight 0.25 in pre_processing_score
Q: Are the GCP coordinates trustworthy as absolute-accuracy anchors?  ·  Owner: GCP processing software + processing path selection  ·  If low: Reprocess via PPK/CORS; require customer accuracy declaration
aggregator: weighted average; GCT1 σ uses mean − 0.25 × (100 − min) across GCPs internally indicators: gcp_sigma_score(0.30), gcp_processing_path_score(0.25), gcp_customer_accuracy_declared_score(0.15), gcp_id_reconciliation_score(0.10), gcp_coord_age_score(0.10), coord_parse_sanity_score(0.05), gcp_residual_score(0.03), cors_station_health_score(0.02)
⊘ gcp_coord_trust_score = 0. Zeros pre_processing_score via block-of-blocks.
survey_design_score
Survey Design Adequacy (GCP-only)
weight 0.1 in pre_processing_score
Q: Was the GCP layout adequate for the site extent?  ·  Owner: Survey planning + operator deployment  ·  If low: Deploy additional GCPs; redistribute; ensure target visibility
aggregator: weighted average of indicator scores indicators: gcp_count_score(0.40), gcp_distribution_score(0.35), target_visibility_score(0.15), vegetation_dtm_declared_score(0.05), software_version_score(0.05)

Level 4 — Derived Fields
Derivations from source fields. Composite-scoring derived fields combine other deriveds.
crs_match_project
Whether declared CRS/datum matches project requirement across all artifacts. Hard-gate input.
= declared_crs_per_artifact == project_required_crs for ALL artifacts
geoid_match_project
Geoid declaration matches project and is internally consistent.
= declared_geoid_per_artifact == project_required_geoid AND consistent across artifacts
height_mode_consistency
Height mode internal consistency + match to project.
= all artifacts declare same height_mode AND matches project_required_height_mode
projection_match_location
Projection sanity check against site coordinates. Hard-gate input.
= declared_projection consistent with survey site bounding box (zone correct)
output_crs_metadata_present
Direct artifact evidence — CRS metadata in EXIF/coord-file headers.
= each output artifact has CRS metadata AND matches declared_crs_per_artifact
units_match_project
Units consistency + match to project.
= declared_units == project_required_units AND consistent
customer_coord_crs_consistent
Applies only on customer-supplied path; N/A otherwise.
= when CUSTOMER_SUPPLIED path: customer_supplied_coord_crs declared AND matches project
localization_disclosed
Fires flag on undisclosed only — not on declared-localized.
= localization_applied_declared IS NOT NULL (operator addressed it, regardless of True/False)
provenance_realization_consistent
All control sources use same ITRF realization and epoch.
= realization_epoch_per_artifact all match
drone_session_within_base_window
PPK base-pairing check.
= (drone_session_start_utc >= base_session_start_utc) AND (drone_session_end_utc <= base_session_end_utc) AND base_file_id matches expected
fraction_geotags_fixed
Fraction of images with FIXED solution. Per-image aggregation.
= (count where per_geotag_fix_status=FIXED) / geotag_count
geotag_completeness_fraction
Fraction of captured images that received geotags.
= geotag_count / captured_image_count
session_overlap_fraction
Fraction of drone flight covered by base/CORS recording.
= (min(drone_end, base_end) - max(drone_start, base_start)) / (drone_end - drone_start)
antenna_pco_match
Antenna PCO declaration vs actual.
= declared_antenna_per_artifact (manifest) == antenna actually used (from device export when available)
sensor_metadata_consistent
Cross-artifact sensor consistency.
= camera_serial (EXIF) == camera declared in manifest == camera in flight log
cors_data_continuity
CORS continuity during flight window — report-dependent.
= cors_epoch_coverage_during_flight from processing report (when uploaded)
time_sync_residual_magnitude
Time-sync drift magnitude.
= statistics from time_sync_residuals (report-dependent)
overlap_texture_proxy
Overlap from manifest; tie-point density when report present.
= min(planned_forward_overlap, planned_side_overlap) AND optional tiepoint_density
flight_conditions_adverse
Declared adverse conditions — cross-handoff to drone chain.
= flight_conditions_declared in {hazy, monsoon, thermal-prone}
gcp_sigma_relative_to_target
Per-GCP σ as multiple of survey accuracy target. Primary trust indicator. Aggregated via mean − 0.25 × (100 − min).
= per_gcp_sigma_h / accuracy_target_m, per-GCP
cp_sigma_relative_to_target
Per-CP σ as multiple of accuracy target. Feeds verification_status.
= per_cp_sigma_h / accuracy_target_m, per-CP
gcp_path_acceptable
AUTONOMOUS path → catastrophic gate.
= declared_path_gcp != AUTONOMOUS
gcp_customer_accuracy_adequate
Customer-supplied accuracy adequacy. Reviewer-blocking when absent.
= when CUSTOMER_SUPPLIED: customer_accuracy_claim present AND meets accuracy_target_m
gcp_id_reconciliation
Cross-artifact ID consistency.
= gcp_ids in coord file match gcp_ids in form/manifest
gcp_coord_age_days
Staleness of customer-supplied GCP coords.
= (flight_date - gcp_coord_determination_date) in days, when CUSTOMER_SUPPLIED
coord_parse_bbox_sanity
Bounding-box sanity check catches axis-swap / unit-misparse.
= parsed gcp_position falls within reconstruction_extent_polygon (with reasonable margin)
gcp_residuals_within_tolerance
GCP residual outlier detection — report-dependent.
= all per_gcp_residuals within tolerance (report-dependent)
cors_station_health_acceptable
CORS station health during observation window.
= cors_quality_metrics (report-dependent)
gcp_count_adequate
GCP count sufficiency.
= gcp_count vs reconstruction_extent_m2 ratio (industry guidance)
gcp_distribution_coverage
Spatial coverage of GCPs vs reconstruction extent.
= fraction of reconstruction_extent_polygon covered by GCP convex hull / grid
target_pixels_at_gsd
Target pixel size for ODM detection.
= target_size_cm / planned_gsd_cm
vegetation_dtm_risk
DTM-on-vegetation risk.
= site_cover_declared=vegetated AND dtm_in_deliverables=True
settings_declared_vs_actual_consistent
Approach 2: validates manifest-declared settings against processing-report-actual settings. Report-dependent.
= when processing report uploaded: report_actual_settings match manifest declarations
software_version_in_buggy_list
v1: advisory only, zero scoring weight. v2: real scoring when list maintained.
= declared_software_version_per_artifact in CBMI known-buggy-versions list (v2)
cp_designated_count
CP count for verification_status determination.
= count of CP records in CP Coord File
cp_distribution_coverage
CP spatial coverage for verification_status.
= fraction of reconstruction_extent_polygon covered by CP convex hull
cp_gcp_spatial_independence
CP-GCP spatial independence for verification_status.
= min pairwise distance from any CP to any GCP

Known limitations (stated, not hidden):
False fix / antenna PCO errors that do not manifest in σ are NOT detectable from artifact evidence alone — caught downstream at processing-stage check-point residuals. Processing-report-dependent indicators (cors_data_continuity, time_sync_residual, gcp_residual, cors_station_health) score as advisory only when report is not uploaded — operational reality is that reports are not reliably uploaded. Settings-actuality validation requires uploaded processing report (Approach 2). Without report, chain validates declared settings only (Approach 1) — operator-declarative misrepresentation not detectable. Approach 3 (independent project-type validation against expected settings) deferred to v2 — requires CBMI to maintain expected-settings reference per project type. Software-version scoring deferred to v2 (SD5) — v1 carries known-buggy-versions flag as advisory only with zero scoring weight; real scoring when CBMI maintains buggy-version list. Approximately 6 of the 42 catalogued problems are NOT detected by pre_processing_score: target washed away (handed to processing_score), stockpile toe ambiguity (handed to volume analytics), monsoon flight artifacts (cross-handoff to drone chain), and others. Documented in problem_coverage_map. Vegetation DTM unreliability is an inherent photogrammetry limitation — chain documents but cannot remove. Undisclosed site localization not detectable from artifact evidence — only detectable when operator addresses it in manifest (declared-only). CRS/projection sanity checks compare against survey bounding box for plausibility; cannot verify absolute correctness when bounding box itself is in wrong frame.

Pre-Processing Closure
Pre-processing now produces five deliverables:
1. pre_processing_score — single survey-level confidence scalar (v1.0 locked)
2. verification_status — categorical CP-verification status (v1.0 locked)
3. geotag_artifact_score — per-artifact diagnostic view (v1.1)
4. gcp_coord_artifact_score — per-artifact diagnostic view (v1.1)
5. cp_coord_artifact_score — per-artifact diagnostic view (v1.1)

Per-downstream-deliverable views (fitness-for-ortho, fitness-for-DTM, etc.) are deferred to a future delivery layer that reads pre-processing's existing indicator values together with processing and analytics indicator values — no further pre-processing changes required.
