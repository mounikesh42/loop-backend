Core — drone_score
drone_score
Drone Quality Score
0.40 * image_capture_score + 0.30 * rover_gnss_score + 0.20 * mission_execution_score + 0.10 * calibration_confidence_score
⊘ GLOBAL GATE (3 chain-level): Any one of the following forces drone_score = 0 (HARD GATE — Q-DRONE-2 lock): (1) L3I_IMG_001 image_validity_score == 0 → CRITICAL_IMAGE_FAILURE (no usable images) (2) L3I_GNSS_001 rover_coverage_score == 0 → RINEX_CRITICAL_GAP (no rover observations during flight) (3) L3I_FC_001 mission_coverage_score == 0 → COVERAGE_GAP (mission area not covered) When any gate fires, overall score is forced to 0. Audit trail must identify which gate(s) fired.
∅ NULL HANDLING (chain-level): drone_score = null when (a) primary source files not provided (raw images, rover RINEX, flight controller log), or (b) all three hard gates would fire (effectively empty survey). Indicator-level null (Q-DRONE-4 new pattern): L3I_FC_005 wind_condition returns null when Open-Meteo API unavailable. Block aggregation treats indicator-null as "not measured" — renormalizes block weight across measured indicators.
∅ INDICATOR-LEVEL NULL (Q-DRONE-4 NEW pattern): Q-DRONE-4 lock v2.1: Drone introduces INDICATOR-LEVEL null (distinct from GCP's chain-level null). When a specific indicator cannot be measured (e.g., wind API unavailable), that indicator returns null rather than fabricating a score. Block aggregation renormalizes across measured indicators only. Distinguishes "we don't know" from "score=0" and from "score=100." Composite scoring at chain and OJS level must respect indicator-null vs zero.
Severity of finding vocabulary (band-level):
Indicator-band level severity. Four values, in increasing severity: NONE (band passed) → MINOR (hygiene signal, audit-only, doesn't drive recommendation) → MATERIAL (warrants review) → CRITICAL (warrants resurvey).
verification_status field (separate categorical):
Separate categorical field reporting drone-level verification quality. VALUES: DRONE_VERIFIED_HIGH (all blocks passed cleanly); DRONE_VERIFIED_MARGINAL (one or more material findings; review_recommended); DRONE_VERIFIED_FAIL (critical finding present; resurvey_recommended); DRONE_HARD_GATE_FIRED (one of three hard gates fired). NEVER score-gating. Score reflects measurement quality; status reports verification basis.
Tier interpretation:
BASE TIER (from numerical score): Gold 90-100; Silver 75-89; Bronze 60-74; Marginal 40-59; Poor <40. HARD GATE: When any of three hard gates fires, drone_score = 0. Tier = Poor (HARD GATE).
