GCP Confidence Score v2 · PPK · Single-Phase
PPK-only single-phase single-scalar redesign. Capture-confidence only — survey-design adequacy (counts, role allocation, target visibility, check-point independence, coordinate accuracy, datum) owned by pre_processing_score. Per-point scoring as primary; device-level rollup as diagnostic context when devices captured multiple points. mean − k·worst aggregation (k=0.25) across GCP-role points. Three building blocks structurally parallel to base_station_score (independently specified).
Computed entirely from GCP device files + flight timing context + external NOAA Kp API. Does not read from base_station_score, drone_score, or pre_processing_score data at runtime.

Core — gcp_score
Single scalar. PPK only. Capture-confidence only. Survey-design adequacy lives in pre_processing_score.
gcp_score
GCP Confidence Score
0.45*gcp_capture_completeness_score + 0.35*gcp_setup_confidence_score + 0.20*gcp_observation_environment_score
per-point: 0.45*per_point_completeness + 0.35*per_point_setup_confidence + 0.20*per_point_environment aggregator: mean − 0.25 × (100 − min) within each role-block, applied across all GCP-role points
⊘ GLOBAL GATE: Sets gcp_score = 0. Raises FLAG: GCP_CRITICAL_FAILURE.
∅ NULL: Zero points with device_role=GCP → gcp_score = null (not 0). FLAG: NO_DESIGNATED_GCPS. Pre-processing handles the survey-design implication.
Hardware-derived capture-confidence score. PPK only. Independent at runtime of base_station_score — requires only GCP device files + flight timing context + NOAA Kp API. Survey-design adequacy (counts, role allocation, check-point independence, target visibility, coordinate accuracy, datum) is owned by pre_processing_score. Check-point scoring is a future deliverable (check_point_score).
Supported device types: CB_X / AEROPOINT / DGPS / OTHER
Level 1 — Building Blocks
Three blocks aggregating per-point category scores across all GCP-role points.
gcp_capture_completeness_score
Capture Completeness & Integrity
weight 0.45 in gcp_score
Q: Are the per-point captures present, complete, and usable?  ·  Owner: Per-point recording / hardware  ·  If low: May need to re-occupy affected points
aggregator: mean − k × (100 − min) across GCP-role per_point_completeness, k=0.25 indicators: occupation_coverage_score(0.35), occupation_integrity_score(0.30), occupation_format_score(0.20), occupation_continuity_score(0.15)
⊘ Triggers GCP_CRITICAL_FAILURE at gcp_score level.
gcp_setup_confidence_score
Per-point Setup & Documentation Confidence
weight 0.35 in gcp_score
Q: Is the per-point operator metadata trustworthy?  ·  Owner: Operator (per-point setup)  ·  If low: Re-measure / re-enter / select correct device on form
aggregator: mean − k × (100 − min) across GCP-role per_point_setup_confidence, k=0.25 indicators: gcp_antenna_height_documented_score(0.55), gcp_device_id_match_score(0.30), gcp_antenna_type_match_score(0.15)
⊘ That point per_point_setup_confidence = 0. FLAG: GCP_POINT_ANTENNA_HEIGHT_MISSING.
gcp_observation_environment_score
Per-point Observation Environment
weight 0.2 in gcp_score
Q: Was the sky/site environment good at each point?  ·  Owner: Site / sky conditions  ·  If low: Relocate next time; accept noise
aggregator: mean − k × (100 − min) across GCP-role per_point_environment, k=0.25 indicators: gcp_multipath_score(0.50), gcp_acquisition_score(0.30), gcp_ionospheric_risk_score(0.20)
Level 2 — Per-point Computation
Each GCP-role point gets its own score using base-v2-shaped formula. Hard gates at per-point level.
per_point_score = 0.45 × per_point_completeness + 0.35 × per_point_setup_confidence + 0.20 × per_point_environment Per-point hard gates: • occupation_coverage_ratio < 1.0 → this point's completeness sub-score = 0 (flag: GCP_POINT_FLIGHT_GAP) • antenna_height_m missing AND device_type ∉ {CB_X, AEROPOINT} → this point's setup sub-score = 0 (flag: GCP_POINT_ANTENNA_HEIGHT_MISSING)
Per-point scores are then aggregated by the building blocks using mean − 0.25 × (100 − min) across all GCP-role points.

Level 4 — Derived Fields
Per-point derivations. Flag-only composites raise warnings; composite-scoring fields feed indicators directly.
occupation_coverage_ratio
Fraction of the drone flight covered by this GCP's observations. Must equal 1.0; less means PPK gaps for this point.
= overlap(obs window, flight window) / duration(flight window)
pre_flight_buffer_sec
Seconds the GCP device was recording before flight start. Threshold >= 120s ideal, >= 60s acceptable.
= flight_start_utc - obs_start_utc
post_flight_buffer_sec
Seconds the GCP device kept recording after flight end. Threshold >= 60s.
= obs_end_utc - flight_end_utc
dual_freq_available
Whether dual-frequency observations are available.
= dual_freq_present == True
cycle_slip_count
Total cycle slips. Elevated = obstruction/disturbance/multipath.
= count of cycle_slip_markers
gap_gt_5s_count
Short recording gaps within the occupation.
= count of inter-epoch gaps > 5s
any_gap_gt_60s
Any major recording gap.
= max inter-epoch gap > 60s
cn0_mean_dbhz
Mean signal strength across the occupation.
= mean of cn0_per_sat
multipath_risk_level
Proxy for multipath risk from a fixed reflector geometry. Risk indicator, not error magnitude. Numeric thresholds: low <= 2.5 dB-Hz; moderate 2.5–4.0; high > 4.0.
= std-dev of cn0_per_sat across sats at equal elevation
mean_pdop
Average sky-geometry quality. Thresholds: <2 = 100; 2-4 = 80; 4-6 = 55; >6 = 30.
= mean of pdop_per_epoch during occupation
max_pdop
Worst-case geometry during occupation.
= max of pdop_per_epoch during occupation
device_acquisition_time_sec
How quickly the device reached a stable solution after power-on. Thresholds: <60s = 100; 60-119s = 88; 120-299s = 65; >=300s = 30.
= time from power-on to stable sat_count (>=6 sats with low PDOP)
rinex_version_supported
Whether RINEX version is accepted by the PPK pipeline.
= rinex_version in {2.10, 2.11, 3.02, 3.03, 3.04, 3.05}
header_completeness
Whether required header records are present.
= antenna_type AND receiver_type AND approx_position_xyz all present
constellation_count
Number of constellations tracked.
= count of constellation_set
session_integrity_ok composite-scoring
Whether the occupation terminated cleanly with adequate power. Device-type aware.
= device_type-aware: for DGPS, requires completed_normally AND shutdowns==0 AND battery_min >= 20%; for CB_X / AEROPOINT / OTHER, oplog expected-absent, evaluates as TRUE if RINEX-side signals are clean (no gap_gt_60s)
antenna_type_match
Form vs RINEX antenna type consistency. NOT ANTEX calibration (Stage 2).
= antenna_model (form) == antenna_type (RINEX), normalised
antenna_height_agreement
Form-height vs RINEX-recorded-height cross-check. Agreement raises confidence; conflict flags the wrong-reference blunder.
= |antenna_height_m (form) - antenna_delta_h (RINEX)| within tolerance (when delta_h != 0)
device_id_match
Form vs RINEX device ID match. Uniform threshold: match=100; mismatch=50 + reviewer-blocking flag (HIGH severity).
= device_id (form) == device_id (RINEX), normalised
kp_index
Geomagnetic activity index. External NOAA SWPC API dependency.
= NOAA SWPC Kp at obs_start_utc
antenna_height_auto_known
Whether antenna height is factory-known for this device type. When true, antenna_height_documented_score = 100 by definition.
= device_type in {CB_X, AEROPOINT}
position_disturbance_signature flag-only
Correlation detector for physical disturbance during occupation (#15). FLAG-ONLY, no weight. BLIND to slow settling that produces no gap/slip.
= gap_gt_5s_count rising AND cycle_slip_count rising AND cn0 variance rising in same time window
truncation_check flag-only
Device-clock vs observation-end cross-check. Disagreement = silent truncation. Applies to DGPS only.
= |session_end_utc - obs_end_utc| within tolerance (when oplog is expected-present)

Level 6 — Source Files
Three per captured GCP-role point. Behaviour branches on device_type.
GCP Device RINEX
.obs / .nav / .21o / .25o
auto
Obtain: Each GCP device records raw observations during its occupation. Downloaded per point either via USB, the CB companion app, the Propeller dashboard (AeroPoint), or the receiver manufacturer's utility (DGPS). Recording must span the drone flight window with buffer.
Per-point machine truth. One file per occupation. Antenna type and delta-H here serve as cross-checks against the user form.
GCP Device Operation Log
.json
auto
Obtain: Device firmware records a session log per occupation. Required for DGPS. Expected-absent for CB_X (CB_X has its own oplog dev spec — see CB_X Operation Log spec). Expected-absent for AEROPOINT (Propeller hardware does not expose one in this format). Expected-absent for OTHER (unknown hardware).
Behaviour branches on device_type. When expected-absent, the integrity indicator scores from RINEX-only signals; when expected-present and missing, integrity degrades to unconfirmed.
GCP User Input (per point)
.json (gcp_user_input.json)
user
Obtain: Structured field form filled per occupation. Human-entered: device_type, antenna model/height/method/reference/count, monument facts, device_id, target metadata. Target size and placement confirmation captured here but consumed by pre_processing_score, not gcp_score.
Per-point operator metadata. Form design itself is the safeguard against #6 antenna-height blunder. target_size_m is on this form but read by pre-processing, not gcp_score.

Known limitations (stated, not hidden):
Per-point disturbance composite is BLIND to slow settling that produces no gap/slip. Antenna-height-missing gate dilutes via aggregation: one bad point in a fleet of 8 reduces gcp_score by ~11 points (vs 35 if it were a single-event score). gcp_set_min_spacing as a #4 partial signal was dropped (Path 1); #4 detection lives in pre_processing_score where the reconstruction extent is known. Operator-honesty-dependent booleans (target_placed_confirmed, etc.) report documentation rigor, not field truth.
