Check Point Confidence Score v1 · RTK · Single-Phase
RTK-only single-phase single-scalar redesign. Capture-confidence only — survey-design adequacy (counts, role allocation, spatial independence from GCPs, reference frame, geoid, base coordinate, processing role assignment) owned by other subsystems. Per-point scoring as primary; sigma is the universal anchor indicator with graceful degradation when missing. Three building blocks structurally parallel to gcp_score and base_station_score (independently specified). mean − k·worst aggregation (k=0.25) across CHECK_POINT-role points.
Computed entirely from check-point RTK device exports + check-point operation logs + check-point user input + flight timing context + external NOAA Kp API. Does not read from base_station_score, gcp_score, drone_score, or pre_processing_score data at runtime.
Source Files (×N per CP point)→Source Fields→Derived Fields→Indicators→Per-point Score→Building Blocks (aggregate)→check_point_score
Core — check_point_score
Single scalar. RTK only. Capture-confidence only. Sigma is the universal anchor.
check_point_score
Check Point Confidence Score
0.45*cp_capture_completeness_score + 0.35*cp_setup_confidence_score + 0.20*cp_observation_environment_score
per-point: 0.45*per_point_completeness + 0.35*per_point_setup_confidence + 0.20*per_point_environment aggregator: mean − 0.25 × (100 − min) within each role-block, applied across all CHECK_POINT-role points
⊘ GLOBAL GATE: Sets check_point_score = 0. Raises FLAG: CP_CRITICAL_FAILURE.
∅ NULL: Zero points with device_role=CHECK_POINT → check_point_score = null (not 0). FLAG: NO_DESIGNATED_CHECK_POINTS. Pre-processing handles the survey-design implication.
Hardware-derived capture-confidence score for RTK check-point captures. Independent at runtime of base_station_score, gcp_score, and pre_processing_score — requires only per-point RTK device exports + check-point operation logs + check-point user input + NOAA Kp API. Survey-design adequacy (counts, role allocation, spatial independence from GCPs, geographic clustering, reference frame, geoid, base coordinate, processing role assignment) is owned by base_station_score, pre_processing_score, or future processing_score and surfaced only there.
Supported device types: CB_X / AEROPOINT / DGPS / OTHER
Level 1 — Building Blocks
Three blocks aggregating per-point category scores across all CHECK_POINT-role points.
cp_capture_completeness_score
Capture Completeness & Integrity
weight 0.45 in check_point_score
Q: Are the per-point captures present, RTK-fix-quality acceptable, and recorded reliably?  ·  Owner: Per-point capture / device  ·  If low: Re-occupy affected points; verify FIXED fix before storing
aggregator: mean − k × (100 − min) across CHECK_POINT-role per_point_completeness, k=0.25 indicators: cp_position_sigma_score(0.45), cp_fix_type_score(0.30), cp_correction_age_score(0.15), cp_log_integrity_score(0.10)
⊘ Triggers CP_CRITICAL_FAILURE at check_point_score level.
cp_setup_confidence_score
Per-point Setup & Documentation Confidence
weight 0.35 in check_point_score
Q: Is per-point operator metadata trustworthy and baseline/mountpoint correctly configured?  ·  Owner: Operator (per-point setup)  ·  If low: Re-measure antenna height; verify mountpoint; bipod or tilt-comp rover
aggregator: mean − k × (100 − min) across CHECK_POINT-role per_point_setup_confidence, k=0.25 indicators: cp_antenna_height_documented_score(0.40), cp_pole_stability_score(0.20), cp_baseline_length_score(0.15), cp_ntrip_mountpoint_score(0.10), cp_antenna_type_match_score(0.10), cp_device_id_match_score(0.05)
⊘ That point per_point_setup_confidence = 0. FLAG: CP_POINT_ANTENNA_HEIGHT_MISSING.
cp_observation_environment_score
Per-point Observation Environment
weight 0.2 in check_point_score
Q: Was the sky and environment good at each capture moment?  ·  Owner: Site / sky conditions  ·  If low: Relocate next time; accept noise; capture at lower-iono time
aggregator: mean − k × (100 − min) across CHECK_POINT-role per_point_environment, k=0.25 indicators: cp_pdop_at_capture_score(0.40), cp_fix_hold_score(0.25), cp_obstruction_score(0.20), cp_ionospheric_risk_score(0.15)
Level 2 — Per-point Computation
Each CHECK_POINT-role point gets its own score using same shape as drone/base/gcp chains. Hard gates at per-point level.
per_point_score = 0.45 × per_point_completeness + 0.35 × per_point_setup_confidence + 0.20 × per_point_environment Per-point hard gates: • fix_type_at_capture ∈ {FLOAT, AUTONOMOUS} → this point's completeness sub-score = 0 (flag: CP_FLOAT_ACCEPTED_AS_FIXED or CP_AUTONOMOUS_ACCEPTED; CRIT or CAT if N<5) • antenna_height_m missing AND device_type ∉ {CB_X, AEROPOINT} → this point's setup sub-score = 0 (flag: CP_POINT_ANTENNA_HEIGHT_MISSING)
Per-point scores are then aggregated by the building blocks using mean − 0.25 × (100 − min) across all CHECK_POINT-role points.

Level 4 — Derived Fields
Per-point derivations. Composite-scoring fields combine other derived fields; flag-only composites raise warnings.
sigma_relative_to_target
Ratio of receiver-estimated horizontal sigma to survey accuracy target. The primary quality indicator. Banded at 1×/2×/5× target.
= position_sigma_horizontal_m / accuracy_target_m
sigma_available
Whether per-point sigma is available in the device export. False triggers either Scenario 1 (exported subset missing sigma) or Scenario 2 (device truly does not report).
= position_sigma_horizontal_m IS NOT NULL
sigma_expected_for_device
Whether sigma should be exported for this device type. If True but sigma is missing → Scenario 1 (fixable by re-export). If False and sigma is missing → Scenario 2 (device limit, N/A treatment).
= device_type IN {CB_X, AEROPOINT, DGPS}
antenna_height_auto_known
Whether antenna height is factory-known for this device type. When True, antenna_height_documented_score = 100 by definition.
= device_type IN {CB_X, AEROPOINT}
antenna_height_agreement
Form-height vs device-reported-height cross-check.
= |antenna_height_m (form) - antenna_delta_h (device)| within tolerance, when both populated
antenna_type_match
Form vs device antenna type consistency.
= antenna_model (form) == antenna_type (device), normalised
device_id_match
Form vs device ID match. Provenance check, not accuracy.
= device_id (form) == device_id (device), normalised
tilt_verifiable
Whether tilt can be verified from device log (vs operator-declarative boolean).
= device_type supports tilt logging AND tilt_logged_deg IS NOT NULL
mountpoint_match
Whether actual NTRIP mountpoint matches project-declared expected mountpoint.
= ntrip_mountpoint == expected_mountpoint
capture_to_flight_delay_hours
Hours between drone flight end and check-point capture. Positive = after flight (preferred).
= (capture_utc - flight_end_utc) / 3600 -- positive if capture after flight
captured_before_flight
True if capture occurred before drone flight start.
= capture_utc < flight_start_utc
captured_during_flight
True if capture occurred during drone flight window.
= flight_start_utc <= capture_utc <= flight_end_utc
kp_index
External geomagnetic activity index. KNOWN LIMITATION: Kp under-detects equatorial scintillation (relevant for India). Long-term roadmap: S4/ROTI from regional reference.
= NOAA SWPC Kp index at capture_utc
dual_freq_available
Whether the device captured dual-frequency data — mitigates ionospheric impact.
= device firmware/model exposes L1+L2 carrier phase
session_integrity_ok composite-scoring
Whether the capture session terminated cleanly with download confirmed and signature (where applicable) intact.
= device_type-aware: CB_X / DGPS / AEROPOINT: completed_normally AND signature valid (if available); OTHER: signature N/A, integrity based on download confirmation
effective_check_point_count
Number of CP points that survived per-point gates. Drives the conditional severity escalation of CP_FLOAT_ACCEPTED_AS_FIXED.
= count of CHECK_POINT-role points where per_point_score > 0

Level 6 — Source Files
Three per captured CHECK_POINT-role point. Behaviour branches on device_type.
RTK Device Export (per point)
.csv / .jxl / .pos / .gpx (vendor-specific)
auto
Obtain: Each RTK check-point capture produces a record in the device export: per-point coordinate, fix_type at capture moment, receiver-estimated sigma (H/V precision), correction age at capture, fix hold duration, PDOP, satellite count, CN0, antenna info. CB_X exports full set; Trimble/Emlid exports subset depending on export config; OTHER variable.
Per-point RTK device record. Replaces RINEX from PPK chains — RTK does not produce extended observation file for post-processing. Per-point sigma is the universal anchor indicator.
Check-Point Operation Log
.json
auto
Obtain: Session-level log: download confirmation, signature/integrity verification, session boundaries. CB_X expected-present with signature; Trimble/Emlid expected-present without signature; OTHER expected-absent.
Behaviour branches on device_type. CB_X is designed to produce signed logs for tamper-evidence; third-party gear typically does not.
Check-Point User Input (per point)
.json (cp_user_input.json)
user
Obtain: Structured field form filled per occupation: device_type, antenna setup, baseline length, NTRIP mountpoint, expected mountpoint, target/mark info, accuracy target. Operator-declarative fields (mark_integrity_confirmed, tilt_compensation_used on non-tilt-comp gear, mark_photo_captured) are advisory only — zero scoring weight per Concern 5.
Per-point operator metadata. Form design itself is the safeguard against antenna-height blunders. Tilt-compensation reads logged tilt verifiably on tilt-comp devices; falls back to advisory boolean on hand-held pole.

Known limitations (stated, not hidden):
False fix (wrong integer ambiguity resolution) is documented as CRITICAL but NOT detected from single-occupation data. Advisory flag CP_NO_REPEATABILITY_CHECK fires on every single-occupation point. Roadmap: when CBMI field workflow adds dual-occupation, repeatability_check_score becomes a real indicator. Sigma may be absent on third-party exports configured to omit precision columns — CP_SIGMA_NOT_EXPORTED flag tells user to re-export. Sigma may be N/A for devices that genuinely do not report it (rare) — weight redistributes within completeness category. Multipath detection is weaker than PPK because RTK captures instantaneous position, not extended observation variance. Kp ionospheric index under-detects equatorial scintillation (relevant for India); long-term roadmap to integrate S4/ROTI from regional reference. 9 catalog rows are dispositioned OWNED_BY_* (other subsystem) and are documented but not detected here. Runtime independence preserved. Antenna-height-missing gate dilutes via aggregation: one bad point in a fleet of N reduces check_point_score by (1/N) × the per-point penalty. Operator-declarative inputs (mark_integrity_confirmed, mark_photo_captured, tilt_compensation_used on non-tilt-comp gear) report documentation rigor, not field truth.
