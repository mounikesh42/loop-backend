Base Station Confidence Score v2 · PPK · Single-Phase
PPK-only single-phase single-scalar redesign. Removed position quality / CORS / known-point / datum / geoid (now pre_processing_score). Three blocks derived top-down from the 22 real-world base station problems. Flag-only composites for cross-signal problems; non-scoring warnings + Learning-Engine handoff for single-session-impossible problems.

Core — base_station_score
Single scalar, computed once at Stage 1, never revised. PPK only.
base_station_score
Base Station Confidence Score
0.45*base_completeness_integrity_score + 0.35*base_setup_confidence_score + 0.20*base_observation_environment_score
⊘ GLOBAL GATE: Sets base_station_score = 0. Raises FLAG: BASE_CRITICAL_FAILURE.
Feeds capture_score alongside drone_score, gcp_score, pre_processing_score. Excludes everything from CORS/known-point/datum/geoid (those live in pre_processing_score). RTK-only concerns out of scope.
Level 1 — Building Blocks
Three independent failure axes. Each can fail while the others are perfect.
base_completeness_integrity_score
Data Completeness & Integrity
weight 0.45 in base_station_score
Q: Is the data there, complete, and usable?  ·  Owner: Recording / hardware  ·  If low: May need to re-survey
0.35*coverage_score + 0.30*integrity_score + 0.20*format_score + 0.15*continuity_score
⊘ Sets block = 0 and base_station_score = 0. FLAG: BASE_RINEX_FLIGHT_GAP + BASE_CRITICAL_FAILURE.
base_setup_confidence_score
Setup & Documentation Confidence
weight 0.35 in base_station_score
Q: Is the operator metadata trustworthy?  ·  Owner: Human (operator)  ·  If low: Re-measure & re-enter; get second-person verification
0.55*antenna_height_documented_score + 0.30*setup_verification_score + 0.15*antenna_type_match_score
⊘ Sets block = 0. FLAG: ANTENNA_HEIGHT_MISSING. (Scores documentation rigor, not measured correctness.)
base_observation_environment_score
Observation Environment Quality
weight 0.2 in base_station_score
Q: Was the sky / site good enough?  ·  Owner: Sky / site  ·  If low: Relocate antenna next time; accept noise or reprocess
0.45*multipath_score + 0.20*ionospheric_risk_score + 0.20*pdop_score + 0.15*acquisition_score

Level 3 — Derived Fields
Computed from source fields. Includes flag-only composites and non-scoring handoffs.
base_flight_coverage_ratio
Fraction of the drone flight covered by base observations. Must equal 1.0; any less means PPK gaps.
= base obs window ∩ flight window / flight window
pre_flight_buffer_sec
Seconds the base was recording before flight start. Threshold ≥120s ideal, ≥60s acceptable.
= flight_start_utc − obs_start_utc
post_flight_buffer_sec
Seconds the base kept recording after flight end. Threshold ≥60s.
= obs_end_utc − flight_end_utc
dual_freq_available
Whether dual-frequency data is available (better ambiguity resolution, iono mitigation).
= dual_freq_present == True
cycle_slip_count
Total cycle slips. Elevated = obstruction/disturbance/multipath.
= count of cycle_slip_markers
gap_gt_5s_count
Short recording gaps.
= count of inter-epoch gaps > 5s
any_gap_gt_60s
Any major recording gap.
= max inter-epoch gap > 60s
cn0_mean_dbhz
Mean signal strength across the session.
= mean of cn0_per_sat
multipath_risk_level
Proxy for multipath from a fixed reflector geometry. Risk indicator, not an error magnitude.
= std-dev of cn0_per_sat across sats at equal elevation
mean_pdop
Average sky-geometry quality.
= mean of pdop_per_epoch during flight window
max_pdop
Worst-case geometry.
= max of pdop_per_epoch during flight window
base_acquisition_time_sec
How quickly the base reached a stable solution.
= time from power-on to stable sat_count
rinex_version_supported
Whether the RINEX version is accepted by the PPK pipeline (#10).
= rinex_version ∈ supported set
header_completeness
Whether antenna type, receiver type, approx position are present in the header (#10).
= required header records all present
constellation_count
Number of constellations the base tracked. Base-side input to #16 cross-check.
= count of constellation_set
session_integrity_ok
Whether the session terminated cleanly with adequate power. Degrades to unconfirmed if Operation Log absent/null.
= completed_normally AND shutdowns==0 AND battery_min adequate
antenna_type_match
Whether the operator-entered antenna model matches the RINEX header type. Type consistency, NOT ANTEX calibration (#21).
= antenna_model == antenna_type (normalised)
antenna_height_agreement
Whether the form height agrees with the RINEX-recorded ARP height. Agreement raises confidence; conflict flags the wrong-reference blunder (#1).
= |antenna_height_m − antenna_delta_h| within tolerance (when delta_h ≠ 0)
kp_index
Geomagnetic activity index. High Kp + single-freq = degraded accuracy (#20). External dependency.
= NOAA SWPC Kp at session time
disturbance_signature flag-only
Correlation detector for physical base disturbance (#7). FLAG-ONLY, no weight. BLIND to slow settling that produces no gap/slip.
= gap_gt_5s_count↑ AND cycle_slip_count↑ AND cn0 variance↑ co-occurring
log_match_check flag-only
Whether the flight falls inside this base log (#15). FLAG-ONLY. Guards wrong multi-flight log matching.
= flight window ⊆ base obs window
truncation_check flag-only
Internal device-clock vs observation end cross-check (#17 cross). Disagreement = silent truncation.
= |session_end_utc − obs_end_utc| within tolerance
autonomous_seed_flag handoff
Records that the base position is an autonomous seed (#3). NON-scoring; judgement deferred to pre-processing.
= approx_position_xyz is autonomous (always true at S1)
benchmark_unverified handoff
Risk signal for unstable/unverified benchmark (#2). NON-scoring warning + monument_id handoff to Learning Engine (#13).
= over_known_mark reused AND NOT verified_by_second_person
Level 4 — Source Fields
Grouped by source file. auto = machine-generated · user = operator-entered.
Base Station RINEX
.obs / .nav / .21o / .25o
marker_name auto
[Header Identity] Station/marker name from RINEX header. Basis for matching the base log to the correct flight.
antenna_type auto
[Header Antenna] Antenna model from the ANT # / TYPE header record. Cross-checked against the operator-entered model.
antenna_delta_h auto
[Header Antenna] ARP height above marker from the ANTENNA: DELTA H/E/N record, as configured in the device. Often 0.000 if unset. Second independent height source.
receiver_type auto
[Header Hardware] Receiver model from the REC # / TYPE / VERS header record.
firmware_version auto
[Header Hardware] Receiver firmware/version where written to the RINEX extended header. Base-side input to the cross-document firmware check (#19). May be null if not emitted.
approx_position_xyz auto
[Header Position] Approximate ECEF position from the header. Always an autonomous seed at Stage 1. Captured as fact only — not scored; raises BASE_AUTONOMOUS_SEED.
rinex_version auto
[Header Format] RINEX format version (e.g. 2.11, 3.04). Drives PPK-software compatibility (#10).
constellation_set auto
[Header Format] Constellations present (GPS/GLONASS/Galileo/BeiDou). Base-side input to the cross-document constellation check (#16).
obs_start_utc auto
[Header Timing] First observation epoch time (UTC).
obs_end_utc auto
[Header Timing] Last observation epoch time (UTC). Cross-checked against Operation Log session_end_utc for truncation.
dual_freq_present auto
[Header Format] Whether L1+L2 (or more) carrier phase is present. Single-frequency limits ambiguity resolution and worsens iono impact.
epoch_interval_sec auto
[Body Timing] Nominal logging interval between epochs.
total_epochs auto
[Body Timing] Count of recorded epochs.
cn0_per_sat auto
[Body Signal] Carrier-to-noise density per satellite per epoch. Basis for signal quality and multipath variance.
cycle_slip_markers auto
[Body Signal] Cycle-slip indications per satellite. Elevated counts indicate disturbance or obstruction.
pdop_per_epoch auto
[Body Geometry] Position dilution of precision over time, from satellite geometry.
sat_count_per_epoch auto
[Body Geometry] Tracked satellite count over time. Acquisition ramp visible at session start.
Operation Log
.json
session_completed_normally auto
[Integrity] True only if recording stopped by deliberate operator action. Null = unconfirmed.
unexpected_shutdown_count auto
[Integrity] Count of mid-session unexpected shutdowns/resets. >=1 raises BASE_SESSION_INTERRUPTED.
battery_start_pct auto
[Power] Battery level at session start (nullable; null for mains/solar).
battery_end_pct auto
[Power] Battery level at session end (nullable).
battery_min_pct auto
[Power] Lowest battery level reached. More important than endpoints — a deep dip means risk even if recovered.
session_end_utc auto
[Integrity] Device-reported session end (UTC). Cross-check partner for RINEX truncation detection.
raw_log_download_confirmed auto
[Integrity] True once the RINEX has actually been transferred off the device. Closes #8.
User Input (Antenna Setup Record)
.json (user_input.json)
antenna_model user
[Antenna Setup] Operator-selected antenna model. Selects the ANTEX/ARP geometry and is cross-checked against RINEX antenna_type.
antenna_height_m user
[Antenna Setup] Operator-entered height. Cross-checked against RINEX antenna_delta_h when present. The single highest-stakes unverifiable value.
antenna_height_units user
[Antenna Setup] Explicit units (m/ft) to prevent unit-mix errors.
antenna_measurement_type user
[Antenna Setup] VERTICAL or SLANT. Slant requires trig conversion and is more error-prone.
measured_to_reference user
[Antenna Setup] Explicit reference point measured to (ARP / slant mark / mount bottom). The field that kills the wrong-reference blunder (#1).
height_measured_count user
[Antenna Setup] Number of independent height measurements/confirmations. Corroboration signal.
pole_tripod_fixed_height user
[Antenna Setup] Fixed pole/tripod height when applicable (known, not re-measured each time).
monument_id user
[Setup Verification] Monument/benchmark identifier. Handed to the Learning Engine for cross-session drift (#2/#13).
over_known_mark user
[Setup Verification] Boolean: was the base set over a known/established mark?
verified_by_second_person user
[Setup Verification] Boolean: was setup independently confirmed?
flight_start_utc user
[Flight Context] Drone flight start (UTC) from the upload bundle. Needed for coverage/buffer (#6).
flight_end_utc user
[Flight Context] Drone flight end (UTC). Needed for coverage and post-flight buffer.
Level 5 — Source Files
The three inputs to the PPK base station confidence score.
Base Station RINEX
.obs / .nav / .21o / .25o
auto
Obtain: Receiver records raw satellite observations continuously from power-on to end of session. Download via USB or the Datum companion app after the survey. Recording must start before drone takeoff and continue past landing.
Machine truth. Self-evidencing for completeness/format/environment. Antenna type and delta-H here serve as cross-checks against the user form.
Operation Log
.json
auto
Obtain: Device firmware records a session log. Exported alongside the RINEX. Scoped (per dev spec) to completeness & power certification only: completed-normally, shutdown count, battery trace, end-time, download-confirmed.
Only source that can certify clean termination. Nullable fields: null = unconfirmed (not bad). If file absent (abrupt death), integrity degrades to unconfirmed, never silently passes.
User Input (Antenna Setup Record)
.json (user_input.json)
user
Obtain: Structured field form. Human-entered: antenna model/height/method/reference/count, monument facts, plus flight start/end UTC from the upload context. The form design itself is the safeguard against the #1 antenna-height blunder.
Human input. Cannot verify correctness, only provenance, corroboration, and consistency with RINEX. Antenna type and height are cross-checked against the RINEX header.

Honest limitations (stated, not hidden):
• #7 slow settling: the disturbance composite catches abrupt disturbance (gap + cycle-slip + CN0 spike) but is blind to a base settling slowly into soft ground with no gap or slip.
• #18 time offset: base-vs-flight UTC/local offset is a cross-document check — not closeable from base data alone.
• #20 ionosphere: requires an external NOAA SWPC Kp-index integration not yet built.
• #21 ANTEX: Stage 1 checks antenna-type consistency only; true ANTEX phase-centre calibration is Stage 2.
• #1 residual: a confidently wrong height that agrees with the RINEX delta-H still passes — only cross-session reconciliation (Learning Engine) or external check can catch it.
