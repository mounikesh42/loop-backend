Base Station Confidence Score v2.1 LOCKED · 3 Blocks · 2 Hard Gates · 4-Severity Vocabulary
Three-block evidence-based base station confidence chain. Confidence-only output: score (0-100) + tier + recommendation. Two HARD gates: coverage gap (L3I_BASE_001=0) and antenna height missing (L3I_BASE_005=0) — both force overall score to 0 (Q1, Q6 locks). Severity vocabulary at band level: none / minor / material / critical. Recommendation vocabulary at chain level: good_to_go / review_recommended / resurvey_recommended. Vocabularies are deliberately separate. Minor findings (per Q4 lock: slow acquisition, log download unconfirmed) are audit-only — do not drive recommendation, visible only in "what passed/what else was checked" expandable per Q5 lock.

Core — base_station_score
base_station_score
Base Station Confidence Score
0.45*base_completeness_integrity_score + 0.35*base_setup_confidence_score + 0.20*base_observation_environment_score
⊘ GLOBAL GATE (2 hard gates): Any one of the following forces base_station_score = 0 (HARD GATE): (1) L3I_BASE_001 coverage_score == 0 (flight gap: BASE_RINEX_FLIGHT_GAP) (2) L3I_BASE_005 antenna_height_documented_score == 0 (no height: ANTENNA_HEIGHT_MISSING) When either fires, overall score is forced to 0 regardless of other indicators. Audit trail must identify which gate(s) fired.
Severity of finding vocabulary (band-level):
Indicator-band level severity (used in operator_action templates and rendering). Four values, in increasing severity: NONE (band passed, no concern) → MINOR (hygiene signal, audit-only; does NOT drive recommendation; visible only in "what passed/what else was checked" expandable per Q5 lock) → MATERIAL (warrants human review before processing; drives review_recommended recommendation) → CRITICAL (serious finding warranting recollection; drives resurvey_recommended recommendation). Note: severity_of_finding is the BAND-LEVEL vocabulary. The CHAIN-LEVEL recommendation vocabulary is good_to_go / review_recommended / resurvey_recommended. These are deliberately separate to prevent conflation (resolved during v2.1 design).
verification_status field (separate categorical):
Separate categorical field reporting base-station-level verification quality. VALUES: BASE_VERIFIED_HIGH (clean session, all blocks passed, no findings); BASE_VERIFIED_MARGINAL (one or more material findings present; review_recommended); BASE_VERIFIED_FAIL (critical finding present; resurvey_recommended); BASE_HARD_GATE_FIRED (one or both hard gates fired — coverage gap or antenna height missing). NEVER score-gating. Score reflects measurement quality; status reports verification basis.
Tier interpretation:
BASE TIER (from numerical score): Gold 90-100; Silver 75-89; Bronze 60-74; Marginal 40-59; Poor <40. HARD GATE: When either hard gate fires (coverage_score = 0 OR antenna_height_documented_score = 0), overall base_station_score is forced to 0. Tier = Poor (HARD GATE). This is enforced at the chain level, not at indicator-band level. Q1-Q6 lock decisions (v2.1): Q1: antenna_height = true hard gate (forces overall score to 0) Q2: RINEX version unsupported = review (not resurvey) — data fixable via conversion Q3: session interrupted stays critical for v2.1; v2 enhancement (flight-window overlap) pending Q4: SLOW_BASE_ACQUISITION and BASE_LOG_DOWNLOAD_UNCONFIRMED = minor (hygiene signals) Q5: minor findings hidden from default customer view, visible only in expandables Q6: global_gate_condition lists both hard gates explicitly
Feeds capture_score alongside drone_score, gcp_score, pre_processing_score. Excludes everything from CORS/known-point/datum/geoid (those live in pre_processing_score). RTK-only concerns out of scope.
