Four-block evidence-based processing confidence chain. Single scalar processing_score per survey. Owns check-point residual analysis as the moment-of-truth survey accuracy measurement. Agisoft Metashape v1 engine; other engines (ODM, Pix4D, RealityCapture) deferred to v2+. Processing report REQUIRED (Option A locked) — processing_score=null when absent because the report IS the evidence base (unlike pre_processing where it is supplementary). verification_status field separates verification-result from quality-score, mirroring pre_processing pattern.

Honest observations on the results:
1. Reports A and B score identically on 4 of 5 views — correct behavior. They produce nearly identical deliverable quality at the per-artifact level; the differentiation (A vs B in chain-level score) comes from statistical sufficiency (CV6) which per-deliverable views deliberately don't emphasize.
2. DTM scores highest because DTM classification (DO4) gets 100 in test setup (DTM not claimed → N/A → 100) and has heavy weight 0.20. If DTM were claimed but unclassified, dtm_score drops sharply.
3. Point cloud and DSM score lowest — depth quality (IM2 at 0.18 / 0.15) drives these views down because reference reports use Medium depth quality.
4. Per-deliverable views correctly capture different concerns for different output types.
